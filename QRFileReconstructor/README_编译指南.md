# 📦 二维码文件重组工具 - 编译与使用指南
## 🛠️ 环境要求
- **Android Studio**: Hedgehog (2023.1.1) 或更高版本
- **JDK**: Java 17 (Android Studio 默认自带)
- **Android SDK**: API Level 34 (Android 14)
- **最低兼容系统**: Android 7.0 (API 24)
## 🚀 快速开始
### 1. 导入项目
1. 打开 Android Studio。
2. 选择 `File` -> `Open`。
3. 选中本目录 (`QRFileReconstructor`) 并点击 `OK`。
4. 等待 Gradle 同步完成（首次同步可能需要下载依赖，请保持网络畅通）。
### 2. 连接设备或启动模拟器
- **真机调试**: 
  - 开启手机“开发者选项”和"USB 调试”。
  - 通过 USB 连接电脑。
- **模拟器**: 
  - 点击 Android Studio 右上角的 `Device Manager` 创建一个新的 Pixel 系列模拟器。
### 3. 运行 App
- 点击绿色运行按钮 (▶️) 或使用快捷键 `Shift + F10`。
- App 安装完成后自动启动。
### 4. 权限授予
- 首次点击“开始扫描”时，App 会请求**相机权限**，请点击“允许”。
### 5. 使用方法
1. **准备数据**: 使用另一台设备显示分割好的二维码图片（JSON 格式）。
2. **开始扫描**: 
   - 点击主界面的 `📷 开始扫描下一张` 按钮。
   - 将摄像头对准二维码。
   - 扫描成功后会自动返回主界面并显示进度（例如：1/7）。
3. **重复操作**: 继续扫描剩余的二维码，直到进度显示 `7/7`。
4. **完成还原**: 
   - 当所有分片收集完毕，App 会自动拼接、解码并校验 MD5。
   - 如果校验成功，会提示“文件还原成功”。
   - 文件保存在手机的 `Download/QRReconstructor/` 目录下。
## ⚠️ 注意事项
- **扫描顺序**: 不需要按顺序扫描，App 会自动根据 `currentPackageNumber` 排序。
- **错误处理**: 如果中途扫错了二维码，可以点击 `🔄 重置任务` 重新开始。
- **存储空间**: 确保手机有足够的存储空间保存还原后的文件。
- **Android 10+**: 文件默认保存到公共下载目录，无需额外配置存储权限。
## 🐛 故障排查
- **无法扫码**: 检查相机权限是否授予，摄像头是否被其他应用占用。
- **校验失败**: 确认所有二维码都清晰且完整，没有扫漏或扫重。如果校验失败，App 会自动重置，请重新扫描所有码。
- **Gradle 同步失败**: 检查网络连接，或尝试在 `File` -> `Invalidate Caches / Restart` 后重试。
